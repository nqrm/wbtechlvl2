Что выведет программа? Объяснить вывод программы. Рассказать про внутреннее устройство слайсов и что происходит при передачи их в качестве аргументов функции.

```go
package main

import (
	"fmt"
)

func main() {
	var s = []string{"1", "2", "3"} // ["1", "2", "3"] len 3, cap 3
	modifySlice(s)
	fmt.Println(s)					// ["3" "2" "3"] len 3, cap 3
}

func modifySlice(i []string) { 		// i = ["1" "2" "3"] len 3, cap 3
	i[0] = "3"						// ["3" "2" "3"] len 3, cap 3
	i = append(i, "4")				// i = ["3" "2" "3", "4", "", ""] len 4, cap 6
	i[1] = "5"						// ["3" "5" "3", "4", "", ""] len 4, cap 6
	i = append(i, "6")				// ["3" "5" "3", "4", "6", ""] len 5, cap 6
}
```

Ответ:
```
Вывод программы: [3 2 3]
Слайс - составная структура данных, которая имеет 3 поля: указатель на базовый массив, длину (length) и вместимость (capacity).
При передаче слайса в качестве агрумента функции вся его внутренняя структура копируется, то есть переменная i содержит тот же слайс, что и переменная s -> тот же базовый массив, те же значения length и capacity.
При вызове функции append() в базовом массиве слайсов i и s не хватает места для добавления нового элемента, поэтому элементы поиндексно копируются в новый базовый массив, который в 2 раза большего предыдущего. Присваивание результата вызова функции append() этой же переменной i приводит к тому, что теперь слайсы i и s содержат разные базовые массивы и измения в них никак не влияют друг на друга.
```
